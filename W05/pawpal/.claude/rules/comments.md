# Comment Guidelines

## When to Comment
- Explain non-obvious logic.
- Clarify algorithms or complex conditions.
- Describe function purpose when it is not immediately clear.

## What to Comment
- Purpose of functions and modules.
- Important assumptions or edge cases.
- Reasoning behind design decisions.

## What Not to Comment
- Avoid explaining obvious code.
- Do not restate what the code already clearly shows.

## Style
- Keep comments concise and clear.
- Write comments in plain language.
- Update comments when code changes.
