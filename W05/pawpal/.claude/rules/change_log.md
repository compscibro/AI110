# Change Log Guidelines

## Purpose
Track all approved changes made to the codebase.

## Rule
- All implemented changes must be recorded in **change_log.txt**.
- Only record changes that were **reviewed and approved** before implementation.

## What to Record
- Bug fixes
- Refactors
- Logic changes
- Test updates
- File modifications

## Format
Date: YYYY-MM-DD  
Files Changed: file_name.py, file_name.py  
Summary: Short explanation of the change and why it was made.

## Example
Date: 2026-03-14  
Files Changed: game_logic.py, test/test_game_logic.py  
Summary: Fixed infinite game loop and corrected score calculation logic.
