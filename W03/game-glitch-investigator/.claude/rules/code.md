# Code Guidelines

## Change Policy
- Do not make unnecessary or radical changes to existing code.
- Preserve the original structure unless a clear issue exists.
- Apply the **smallest possible modification** needed to fix a bug or improve clarity.
- Refactor only when there is redundant, duplicated, or clearly problematic code.

## Readability
- Keep code clean and easy to understand.
- Use clear and descriptive variable and function names.
- Prefer simple and straightforward logic.

## Modularity
- Maintain modular structure where possible.
- Keep functions focused on a single responsibility.
- Avoid tightly coupled code between modules.

## Style
- Follow Python best practices and PEP 8 guidelines.
- Keep formatting consistent across files.

## AI Assistance
- AI-generated suggestions must be reviewed before implementation.
- Changes should prioritize correctness, readability, and minimal disruption to the existing codebase.
